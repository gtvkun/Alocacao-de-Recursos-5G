O Desafio: Alocação de Recursos em Redes 5G

Redes 5G precisam atender a uma enorme diversidade de aplicações, desde streaming de vídeo (alta vazão) até serviços de emergência (baixa latência). Alocar os recursos da rede (RBs - Blocos de Recursos) de forma eficiente é um problema complexo.

O Objetivo Principal:

Encontrar um equilíbrio ideal entre dois objetivos concorrentes:

  • Máxima Satisfação do Usuário: Atender às necessidades de QoS (Qualidade de Serviço) de cada aplicação.
  • Justiça (Fairness): Garantir que nenhum usuário seja deixado de lado (evitar "starvation"), distribuindo os recursos de forma equitativa.

O Que é um Algoritmo Genético (AG)?

É uma técnica de otimização inspirada na evolução biológica de Charles Darwin. O AG "evolui" uma população de soluções candidatas em direção a uma solução ótima.

População: Conjunto de possíveis soluções.
Fitness: Nota que avalia quão boa é uma solução.
Seleção: Os mais "aptos" são escolhidos para reproduzir.
Cruzamento (Crossover): Pais combinam "genes" para criar filhos.
Mutação: Pequenas alterações aleatórias para criar diversidade.
Elitismo: Os melhores indivíduos sobrevivem intactos.

AG: Representação do Problema

O Indivíduo (ou Cromossomo)

Cada indivíduo é uma solução completa para o problema. Neste caso, é uma lista que representa a alocação de RBs para todos os usuários.

[ 12, 8, 0, 5, ..., 15 ]

Onde o índice é o ID do usuário e o valor é a quantidade de RBs alocados.

A População

A população é um conjunto desses indivíduos. Cada um representa uma estratégia de alocação diferente. O AG trabalha com múltiplas soluções ao mesmo tempo.

Indivíduo 1: [ 12, 8, 0, ... ]
Indivíduo 2: [ 5, 10, 4, ... ]
Indivíduo 3: [ 0, 7, 11, ... ]
...

O Coração do AG: A Função de Aptidão (Fitness)

A função fit() em 5g_v2.py determina a "qualidade" de cada alocação. É uma função multi-objetivo ponderada:

Fitness = (0.7 * Satisfação Média) + (0.3 * Índice de Justiça de Jain)

Satisfação

Mede o quão bem a alocação atende aos requisitos de cada aplicação (eMBB vs URLLC). Uma função "suave" que guia a evolução mesmo para soluções subótimas.

Justiça (Fairness)

Usa o Índice de Justiça de Jain, uma métrica padrão para medir a equidade. Um valor de 1 indica justiça perfeita.

Operadores Genéticos: Cruzamento (Crossover)

A função cross() combina dois "pais" para gerar um novo "filho", explorando novas soluções.

Usa-se o cruzamento de ponto único. O mais importante é o mecanismo de reparação.

Pai 1: [ 10, 5, 8 | 12, 3, 7 ]
Pai 2: [ 4, 9, 11 | 6, 2, 1 ]

Filho: [ 10, 5, 8, 6, 2, 1 ]

Operadores Genéticos: Mutação e Elitismo

Mutação Híbrida

A função mutate() introduz diversidade para evitar estagnação:

  1. Tentativa Guiada: Move recursos de usuários satisfeitos para usuários necessitados. (Refinamento)
  2. Fallback Aleatório: Se a guiada falhar, troca recursos entre dois usuários aleatórios. (Exploração)

Elitismo

Garante que as melhores soluções encontradas não sejam perdidas. Os melhores indivíduos (ex: 2 melhores) de uma geração são copiados diretamente para a próxima, preservando o progresso.

Com Quem Estamos Comparando? (Baselines)

Para avaliar o AG, seu desempenho é comparado com algoritmos clássicos:

Round Robin (RR)

Divide os recursos igualmente. É muito justo, mas ineficiente, pois ignora a qualidade do sinal e as necessidades dos usuários.

Best CQI (Greedy)

Aloca tudo para os usuários com melhor qualidade de sinal (CQI). Maximiza a vazão, mas é extremamente injusto.

Proportional Fair (PF)

Busca um equilíbrio entre a justiça do RR e a eficiência do Best CQI. É um competidor forte e amplamente utilizado.
